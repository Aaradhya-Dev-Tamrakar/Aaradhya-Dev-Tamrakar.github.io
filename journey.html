16
  • The VIP/Master tier gate only toggled href/CSS on each project's anchor tag client-side — the real GitHub URL still sat as plaintext in a data-vip-link attribute for all 13 projects, readable by anyone viewing raw HTML, curling the page, or hitting a search-engine cache, passcode or not.
  • Moved every project repo URL through the same AES-256-GCM/PBKDF2 pipeline already used for gated page content — data-payload-link-id references now stand in for the raw href, decrypted at runtime via decryptHexPayload() only after tier verification passes.
  • Follow-up pass removed the login vectors that didn't belong on a public tier system in the first place — a UI tab, a keyboard shortcut, and two undocumented trigger sequences — leaving Google Sign-In against an explicit allowlist as the only path to Master tier.
